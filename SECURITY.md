# Security Policy

## Reporting a Vulnerability

We take the security of our project seriously. If you believe you have found a security vulnerability, please report it to us as described below.

### How to Report

Please **do not** create public GitHub issues for security vulnerabilities. Instead, we ask that you privately disclose your findings by:

1. Sending an email to `security@example.com` (replace with actual contact)
2. Or opening a private advisory via the [GitHub Security Advisories](https://github.com/your-org/your-repo/security/advisories/new) feature if available

### What to Include

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps to reproduce the problem
- Any relevant code snippets or configuration details
- Your assessment of potential impact (if known)

### Response Timeline

We aim to acknowledge receipt of your report within **48 hours** and will provide regular updates on our progress. We typically resolve critical security issues within **14 days**.

## Security Best Practices

Users are encouraged to:

- Keep the project updated to the latest version
- Follow secure configuration guidelines in the documentation
- Use strong authentication credentials where applicable
- Regularly review access logs and audit trails

## Supported Versions

Security updates are only provided for the current stable release. Older versions may not receive security patches.

| Latest  | :white_check_mark: |
